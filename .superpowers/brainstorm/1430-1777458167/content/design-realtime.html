Design: Live Data + Real-Time Updates

The app already has a battle-tested Supabase Realtime hook. We extend it — no new infrastructure needed.

What Triggers a Live Update

Student asks a question → /api/weak-topics POST runs → inserts/updates weak_topics in DB → Supabase Realtime fires → debounced fetchWeakTopics() → WeakTopicCards re-render within ~500ms
Student adds an exam → /api/exam POST → inserts into exams table → Realtime fires → debounced fetchExam() → ExamCountdownSection + WeakTopicsSection re-filter instantly
Exam expires (date passes) → PATCH to status="completed" → Realtime fires on exams → fetchExam() re-runs → expired exam removed from activeExams → selectedExam auto-resets to nearest valid exam
Student completes a quiz → quiz results update mastery → weak_topics count may decrease → Realtime fires → fetchWeakTopics() → topic disappears from weak list if below threshold
Tab hidden → visible again → existing visibilitychange handler already pauses and resumes → on tab focus, guardedRefetch() fires both fetchExam + fetchWeakTopics immediately
WebSocket disconnects → existing fallback kicks in → 60s polling loop calls fetchExam + fetchWeakTopics until Realtime reconnects. Status indicator shows "polling" mode.
