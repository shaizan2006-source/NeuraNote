Crash-Proof: Error Handling Strategy

React Error Boundary wraps entire ExamCard — crashes in any child show a graceful fallback, not a blank screen
Double normalization Both exam.subject and weakTopic.subject run through normalizeSubject() before comparison — handles "DBMS", "dbms", "Database Management" all matching
Null-subject fallback Old exams with null subject → show all topics (no filter) with a banner: "Subject not set — showing all weak topics"
sessionStorage try/catch All sessionStorage reads/writes wrapped in try/catch — private browsing mode blocks it silently. Falls back gracefully (quiz opens without prefill)
Fetch retry with backoff fetchWeakTopics and fetchExam retry up to 2x with exponential backoff before showing the error state with Retry button
Race condition guard fetchWeakTopics uses a ref-based in-flight guard (same pattern as fetchProgressSummary already in context) — no double-fetches
Deleted exam → reset When selectedExam is removed from activeExams, context auto-selects nearest remaining exam or sets null → WeakTopicsSection reacts immediately
Invalid date guard If exam_date produces NaN — already handled in ExamCountdownSection. Study plan generation checks daysLeft ≥ 0 before distributing topics
Study plan edge cases 0 days left → "Exam is today, focus on quick revision". 1 day → single-day plan. More topics than days → stack 2–3 topics per day. No topics → show generic revision checklist
Accuracy fallback WeakTopics from DB only have count, not an accuracy %. Accuracy is derived: accuracy = max(0, 100 - (count * 12)) capped at 59% (since threshold=5 means ≥5 errors). Displayed on progress bar.
