Design: Full Live Feedback Loop

Critical finding: quiz completion currently does NOTHING to the DB. Here's the complete fix.

What Actually Happens Today vs What Needs to Happen

Current — Dead Loop
1. Student takes quiz on "normalization"
2. Gets 8/10 correct — performance tracked in local state only
3. handleFinish() → clearSession() → router.push('/dashboard')
❌ ZERO API calls. Nothing written to DB.
4. Dashboard loads — "normalization" still shows as weak ❌
mastery_topics unchanged ❌
weak_topics count unchanged ❌
After Fix — Full Live Loop
1. Student takes quiz on "normalization"
2. Gets 8/10 correct — signals: {topic, correct, total, subject}
3. handleFinish() → POST /api/quiz-results →
✅ mastery_topics upserted
✅ weak_topics count reduced / removed
✅ learning_events logged
4. Supabase Realtime fires on both tables
✅ fetchWeakTopics() triggers
✅ "normalization" card disappears or accuracy jumps
✅ Progress Snapshot updates — live, within ~1s

New: POST /api/quiz-results — The Missing Link

Input: 
{ userId, subject, results: [{ topic, correct, total }] }
Logic per topic:
  • Call existing updateMastery({ user_id, topic, correct, total })
  • accuracy = correct/total * 100
  • If accuracy ≥ 80% → DELETE from weak_topics (topic mastered)
  • If accuracy ≥ 60% → UPDATE weak_topics SET count = max(count - 2, 0), level = "medium"
  • If accuracy < 60% → UPDATE weak_topics SET count = count + 1, level = "hard" (still weak)
  • INSERT into learning_events: { event_type: "quiz_completed", topic, accuracy, subject }
Result: Realtime fires on mastery_topics + weak_topics → dashboard updates live

Complete Event → Dashboard Update Map

→ /api/weak-topics POST → weak_topics upsert → Realtime → fetchWeakTopics() → WeakTopicCard appears/count increases

Student completes quiz ~1s update on return

→ /api/quiz-results POST → mastery_topics + weak_topics updated → Realtime → fetchWeakTopics() + fetchMastery() → cards update/disappear, accuracy bar jumps, Progress Snapshot refreshes

Student completes focus task ~1s update

→ markTaskDone() already POSTs to /api/focus-progress → focus_progress already in Realtime TABLES → fetchProgressSummary() → Progress Snapshot "last active" updates

Student adds/removes exam ~1s update

→ exams already in Realtime TABLES + we add refetchExam callback → ExamCountdownSection + WeakTopicsSection re-filter for new subject instantly

Exam countdown ticks every second

→ setInterval every 1s already in ExamCountdownSection → days/urgency color updates live. On day change → urgency threshold re-evaluates automatically.

Tab comes back to focus instant

→ visibilitychange handler re-fires guardedRefetch() + sets up Realtime channel again → fetchExam + fetchWeakTopics fire immediately → everything syncs on tab return

Progress Snapshot — What Each Stat Reflects Live

Avg Accuracy

Derived from masteryTopics filtered by subject. Updates when mastery_topics Realtime fires after quiz. Formula: avg(mastery_score) for subject topics.

Topics Mastered

masteryTopics where mastery_score ≥ 70, filtered by subject. Numerator/denominator both update via Realtime after quiz completion.

Last Active

From streak/lastActiveDate already in context. Updates after any session (ask, quiz, focus) via fetchStreak() called in fireTracking().
