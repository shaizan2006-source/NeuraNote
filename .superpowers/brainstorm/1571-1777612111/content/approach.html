Dashboard Rebuild: Which Approach?

Your codebase already has all 5 dashboard cards in JS. How should we get to the pixel-perfect image?

A

Enhance Existing (Recommended)

Restyle current components in-place — AskAIHeroCard, StudyModeCards, DashboardSidebar, GreetingRow — to pixel-match the image. Keeps all backend wiring (Supabase, DashboardContext, APIs) intact. Fastest path, zero regressions.

Pros

  • All APIs already wired
  • No data migration
  • Ship fastest

Cons

  • JS, not TS
  • Inline styles (no Tailwind)
B

Full TypeScript Rebuild

Brand-new .tsx components with Tailwind classes, Zustand store, and typed API calls — replacing the existing .jsx files. Exactly what the spec describes (TypeScript + Tailwind + Zustand).

Pros

  • Clean TypeScript
  • Tailwind-first
  • Zustand state

Cons

  • Must re-wire all APIs
  • Risk of regressions
  • Longer timeline
C

Hybrid: Keep Logic, New Styles

Keep DashboardContext + API hooks unchanged. Convert just the visual layer — swap inline styles for Tailwind classes, add Framer Motion polish, match the image's glassmorphism system.

Pros

  • Best visual output
  • APIs stay intact
  • Tailwind design system

Cons

  • Requires Tailwind config
  • Medium effort
