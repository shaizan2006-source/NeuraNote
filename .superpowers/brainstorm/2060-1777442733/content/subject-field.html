How should Exams link to Subjects?

This is the foundation of the whole filtering system — weak topics are filtered by subject. The current modal only has exam name + date. Which approach should we use?

A

Add a separate Subject field to the modal

Student enters Exam Name (e.g. "JEE Main 2025") and separately picks a Subject from a dropdown (Physics, Chemistry, Maths, DBMS, OS, etc). Weak topics are then matched exactly to the chosen subject.

Pros

  • Precise matching — no guesswork
  • Multi-subject exams handled correctly
  • Clean subject filtering logic

Cons

  • Requires DB schema change (add subject column to exams table)
  • More fields in the add-exam modal
Add Exam Modal — Option A
B

Derive subject from exam name (current approach + better matching)

Keep the modal as-is (name + date). Improve normalizeSubject() to infer the subject from the exam name. E.g. "JEE Main" → "jee main", "DBMS Mid-term" → "dbms". Weak topics match against the normalized key.

Pros

  • No schema changes needed
  • Modal stays clean — 2 fields only
  • Works with existing normalizeSubject() map

Cons

  • "JEE Main" doesn't match weak topic subjects like "physics" or "chemistry"
  • Multi-subject exams are ambiguous
  • Relies on naming conventions
Add Exam Modal — Option B (unchanged)
