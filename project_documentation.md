# Ask My Notes - Complete Project Documentation

## 1. Product Overview & Target Audience
**Ask My Notes** is an AI-powered SaaS application purpose-built to act as a study companion for university level students (B.Tech, BCA, BSc, Medical, Law, Finance, BBA, MBA, MBBS, and other university departments). It transforms passive reading of notes and PDFs into an active, adaptive learning experience through interactive Q&A, AI coaching, voice tutoring, and personalized adaptive study plans.

## 2. Core Features & Functionality

### 2.1 PDF Upload & Contextual Q&A (RAG)
- **Upload Pipeline:** Users can upload PDF study materials. The system extracts the text using `pdf-parse`, chunk it into manageable pieces (via LangChain), and generates vector embeddings using OpenAI's `text-embedding-3-small`. These embeddings are stored in Supabase using `pgvector`.
- **Ask AI:** Users can chat with their documents. When a question is asked, the system performs a vector similarity search across the document chunks, fetching the top 5 most relevant sections. It then prompts OpenAI `gpt-4o-mini` to stream back an answer, complete with citation sources.
- **Exporting Options:** Answers generated by the AI can be exported to fully formatted PDF or DOCX documents for offline use.

### 2.2 Adaptive Study Planner & Gamification
- **Daily Plan:** A dynamic, priority-based study planner that tells the student exactly what to focus on today. Priority is calculated algorithmically based on:
  1. Weak topics
  2. Exam proximity
  3. Spaced Repetition (SRS) schedules
  4. Memory/Interests
- **Streak Tracking & Analytics:** Motivates students by tracking daily question streaks. A weekly recap breaks down questions asked and quizzes taken.
- **Focus Mode:** A built-in study timer (Pomodoro style) with an SVG radial progress ring, tracking focused sessions and task completions.

### 2.3 Mastery Tracking & Quiz Generation
- **Weak Topic Extraction:** As students ask questions, the system extracts the underlying topic and tracks repeated queries to identify "weak topics" the user struggles with.
- **AI Quizzes:** The system can autonomously generate 5-question multiple-choice quizzes targeting the user's identified weak areas, keeping them challenged.
- **Brain Map:** A visual dashboard component that maps out the student's mastery of topics. Topics are rated continuously based on quiz scores and spaced-repetition levels (color-coded Red, Orange, Green).

### 2.4 Multi-modal AI Coaching
- **Text Coach:** A dedicated chat interface for broader study advice, motivation, and abstract topic explanations (distinct from document Q&A).
- **Voice AI Tutor:** An interactive voice-call feature simulating a real tutor. It takes user microphone input, uses OpenAI Whisper for transcription, formulates a response using a Feynman-style teaching prompt (simple to deep), and reads it back via TTS (Text-to-Speech).

### 2.5 Monetization (Razorpay)
The platform is gated behind tier limits using Razorpay:
- **Free:** 1 PDF, 20 Q&As per day, 10 min Voice limit.
- **Student (₹299):** 10 PDFs, unlimited Q&A, 20 min Voice limit.
- **Pro (₹599):** Unlimited PDFs & Q&A, 40 min Voice limit.
- **School (₹50,000):** Unlimited everything (enterprise).

## 3. Technology Stack Architecture
Ask My Notes is built for high speed, edge capability, and high performance using a modern React & Next.js stack.

- **Frontend:** Next.js 16 (App Router), React 19, Tailwind CSS v4, Framer Motion (for polished, fluid animations), Recharts (for analytics dashboards), Shadcn UI.
- **Backend/API:** Next.js serverless API routes hosted on Vercel.
- **Database:** Supabase (PostgreSQL). Stores users, document chunks, mastery tracking, and chat histories. Uses `pgvector` for semantic document search.
- **Authentication:** Supabase Auth (Email/Password & standard flows), fully integrated into the client DashboardContext and API headers via JWT barrier tokens.
- **AI Tooling:** OpenAI models (`gpt-4o-mini`, `gpt-4o`, `text-embedding-3-small`, Whisper STT, TTS), LangChain Core for text segmentation, Tesseract.js (for potential OCR workflows).

## 5. End-to-End User Flow
1. **Acquisition:** User lands on the animated landing page and clicks "Get Started".
2. **Onboarding:** Registers via Supabase Auth. Fills an onboarding form specifying their target course/semester (e.g., B.Tech Final Exams) and target exam date.
3. **Main Dashboard:** 
   - They load into the **Study** tab to see their active daily plan and exam countdown. 
   - They upload a reference PDF, watch the processing bar, and start chatting with it instantly via the "Ask AI" component.
   - If they get confused, they jump into a simulated voice call with the AI Tutor.
4. **Practice & Analytics:**
   - Over a few days, the app detects they struggle with specific physics concepts.
   - They navigate to the **Practice** tab, click "Generate Quiz", and are tested on those weak areas.
   - They view their progress and daily answering streak via interactive charts in the **Analyze** tab, and their "Brain Map" starts turning from Red to Green.
5. **Conversion:** They hit the 20-question free daily limit and are presented with an upgrade prompt, smoothly taking them to the Razorpay checkout to purchase the "Pro" tier.
